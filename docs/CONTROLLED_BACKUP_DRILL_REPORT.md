## 5. Contract Gaps Found and Fixed

### Gap 1 — Missing status markers
- **Problem**: `scheduled-backup.ps1` did not create `status.success` or `status.failed` after execution.
- **Impact**: `verify-latest-backup.ps1` and `cleanup-old-backups.ps1` aborted safely (correct behavior), but the backup set could not be verified or cleaned up.
- **Fix**: Added `Write-StatusFailed` function and `status.success` writer with mutual exclusion logic.
- **Commit**: `5ab2cb2` — `fix: write backup status markers`
- **Version bump**: `1.0.0 → 1.1.0`

### Gap 2 — Missing manifest.sha256
- **Problem**: `verify-latest-backup.ps1` requires a `manifest.sha256` file, but no script generated one.
- **Impact**: Checksum verification step failed with `Checksum file missing`.
- **Fix**: Added `Generate-Manifest` function in `scheduled-backup.ps1` to hash all data files using SHA256 with relative forward-slash paths, placed between MinIO backup and `status.success` write.
- **Commit**: `840c1c1` — `fix: generate backup manifest checksum`
- **Version bump**: `1.1.0 → 1.2.0`
